Idle Power City

Big Bang Studio Ltd.

Genre: Idle Arcade, Hybrid-Casual, City Builder Platform: Android Role: Game Developer Technology Used: Unity, C# Soft Launched by: MondayOff Short Description: Idle Power city is a 3d idle arcade mobile game where the players are required to build a city and keep it running by collecting and dropping power resources.   • I implemented:
    • Core Gameplay Systems:-
      • Player Behavior-
        • Joystick based top-down all directional movement system.
        • Player can interact with the world using different triggers and colliders.
        • Player can pick up energy cells, fuels and rocket fuels, drop them, and interact with buildings. Made a custom bezier curve system for the collection and dropping arc.
        • Player has a limited but upgradable number of stack capacity and movement speed.
        • The main goal for the player is to unlock different areas and grow the city.
      • Building System-
        • Building unlock and upgrade system with cash. The HR building and the ship doc will unlock automatically.
        • Upgrading each building gives 2 options for the visuals. A free option and a premium option which can be redeemed with RV.
        • Each resource building will generate different resource items such as Energy cells, Fuels, Rocket Fuels etc.
        • All other buildings have npcs that will react to the buildings being active/deactive.
        • Buildings are needed to be activated by the player or helper npcs using the cells or fuels to generate money.
        • Shops have a service npc inside that will serve the produced foods. Each ingrident dropped into the building will produce a specific number of food.
        • Each building will produce cash based on the time they are active.
        • There HR building is an upgrade buiding for the player to upgrade character stats, hire npc helpers and upgrade their stats too.
      • Mission/Quest System-
        • The mission system is there to guide the players and give them clear goals.
        • Each mission gives some rewards to the player and completing one or more missions will make the player be able to unlock the next building or next task.
        • Daily missions unlocks after completing a certain number of missions.
        • Three Daily missions are randomly generated. Completing each mission yields a reward and completing all three gives a grand reward.
        • Daily missions reset every 24 hours while the main missions never resets.
    • AI Systems:-
      • All the AI were created using a custom Finite State Machine System.
      • All the AI that has movement have a navmesh agent component with different settings each while the static ones don't have any navmesh. The navmesh surface is same for all moving NPCs (Helpers and Shop customers).
      • Helper Npc-
        • Helper npcs are hireable from the player upgrade building. They will help the player manage the farm and gather more money.
        • Helper npcs have Idle, Wandering, Going towards resource building, Collecting, Going to delivery building, and Dropping states.
        • The npcs have 2 major behaviors, collecting resources and dropping items.
        • During the wandering state the npc will select a delivery building which is inactive. If all the buildings are active then randomly select one. After selecting the delivery building the npc then selects the corresponding resource building and changes the state to Going towards resource building.
        • The collection of items were handled to prioritize the player to handle errors.
        • Once collection is done (finite stack capacity), the npc state is then Going to delivery building. After reaching the building the npc will drop the resources, and restart the cycle.
        • Helper npcs have limited but upgradable number of speed and stack capacity and can be hired a limited number.
      • Customer Npc-
        • The customers (both humans and boats) have Idle, Wandering, Going towards shop, Waiting to Collect, Collecting, Moving to despawn area.
        • There is a spawning system for the customers where each shop can hold a limited number of queue.
        • At the begining of a customer life cycle, they will choose what to order based on available shops. Each customer may order multiple items and move to multiple shops.
        • The boat will order a certain energy source.
      • Misc Npcs-
        • These are all the npcs that are inside the buildings that require activation.
        • Each npc has different behaviors and will work accordingly.
        • Static npcs will not move. Moving npcs will move inside a certain area.
        • The train station has a simulation of train coming and going and npcs moving in and out of it.
        • The gas station has cars as npcs. The car behaviors were done using Unity splines
        • The factory has a truck and a worker as npc. The factory generates some items which are carried by the worker to the truck. Once the truck is filled it goes away and another truck arrives.
        • The rocket station has no npc. The rocket moves and stays out for a certain amount of time.
    • UI Systems:-
      • Popups-
        • All sorts of popups including settings, upgrade and hire popup, ads or IAP popup etc.
      • World UI-
        • Unlock, Upgrade, Collect and, Drop area UIs for player interactions.
    • Monetization Systems:-
      • Ads-
        • Banner, Interstital and, Rewarded ads were implemented using the max sdk.
        • There are some Rewarded video items placed on the playing field to give extra boost to the players. The rewards include cash, items and increased player speed, capacity and the production time of the buildings.
        • Each resource building also has an ad button to get many items faster.
        • Building upgrades has a premium visual options that can be unlocked using RV ads.
        • The ads were placed in a way that it doesn't disturb the player experience.
      • In-App Purchases-
        • Only the ads removal IAP was implemented.
    • Level Design:-
      • The levels are divided into 6 different unlockable areas. Each area has a somewhat different theme and different buildings.
      • Completing a certain number of missions prompted the next area to be unlocked which then had to be unlocked with currency and resources.
      • The buildings and props were placed in each of the area manually.
    • Misc:-
      • Custom water shader using the Shader Graph for the entire map.
      • Custom behind the wall glow shader using the Shader Graph for better visuals.
      • Lighting and camera controls.
      • Particles and FX.
